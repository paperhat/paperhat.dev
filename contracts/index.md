# Paperhat Contracts

This index explains why Paperhat has **Contracts**, and provides stable links to the active contract documents.

Contracts define **exclusive responsibilities**, **authority boundaries**, and **non-negotiable invariants** for the Paperhat system and its libraries.

---

## 1. Purpose

This index exists to:

* explain the role of Contracts in Paperhat documentation
* provide a single click-through entry point for contract browsing on GH Pages
* preserve the SCREAMING_SNAKE_CASE contract naming convention

---

## 2. What a Contract Is (Normative)

A Paperhat Contract is a **normative document** that:

* defines exclusive ownership of responsibilities
* defines required boundaries (what a library/document MUST NOT do)
* defines invariants other documents MUST uphold
* prevents semantic drift and accidental framework behavior
